Vamos simular o provisionamento dos dois serviços de dados mais críticos usando a linha de comando.

## Seção 1: Gerenciamento de Arquivos no S3 (Armazenamento de Objetos)
Vamos criar um *bucket* para armazenar arquivos de configuração e provar o *upload* a partir do WSL.

1.  Crie o diretório de trabalho do laboratório.
```bash
mkdir -p ~/aulas_lab/aula011
cd ~/aulas_lab/aula011
```

2.  **Criação do Arquivo Local:** Crie um arquivo de configuração de teste no seu terminal WSL.
```bash
touch config_app_v1.txt
echo "DB_HOST=localhost" > config_app_v1.txt
```

3.  **Definir Nome do Bucket:** O nome deve ser **globalmente único** e em minúsculas. Use seu nome ou um identificador único.
```bash
BUCKET_NAME="lab-devops-12-unifaat" 
```

3.  **Criar o Bucket:** Crie o *bucket* na sua região padrão.
```bash
aws s3 mb s3://$BUCKET_NAME
```

4.  **Upload do Arquivo:** Copie o arquivo local para o *bucket* usando o comando `cp`.
```bash
aws s3 cp config_app_v1.txt s3://$BUCKET_NAME/config/
```

5.  **Verificação:** Liste o conteúdo do *bucket* (e o subdiretório `config/`) para confirmar o *upload*.
```bash
aws s3 ls s3://$BUCKET_NAME/config/
```

6.  **Limpeza do S3:** Remova o arquivo e depois o *bucket* (o *bucket* deve estar vazio para ser removido).
```bash
aws s3 rm s3://$BUCKET_NAME/config/config_app_v1.txt
aws s3 rb s3://$BUCKET_NAME
```

## Seção 2: Simulação de Criação de RDS (Database Gerenciado)
A criação de um RDS real é complexa e demorada. Usaremos o comando `aws rds describe-db-instances` para simular a inspeção e entender o potencial da CLI.

1.  **Pré-requisito:** Obtenha o ID de uma **VPC** existente na sua conta, pois o RDS precisa ser lançado dentro de uma VPC.
```bash
VPC_ID=$(aws ec2 describe-vpcs --query "Vpcs[0].VpcId" --output text)
echo "VPC ID Selecionada: $VPC_ID"
```

2.  **Criação da Instância RDS PostgreSQL:** Execute os comandos abaixo para criar um RDS PostgreSQL gerenciado.

```bash
# Definir variáveis
DB_IDENTIFIER="devopsdb-postgres"
DB_MASTER_USER="postgres"
DB_PASSWORD="UniFAAT2026"
DB_INSTANCE_CLASS="db.t3.micro"
REGION="us-east-2"

# Obter informações da VPC e Security Group (execute um de cada vez)
VPC_ID=$(aws ec2 describe-vpcs --query "Vpcs[0].VpcId" --output text --region $REGION)
echo "VPC ID: $VPC_ID"

# Obter um Security Group existente ou criar um
SG_ID=$(aws ec2 describe-security-groups --filters "Name=vpc-id,Values=$VPC_ID" --query "SecurityGroups[0].GroupId" --output text --region $REGION)
echo "Security Group ID: $SG_ID"

# Descobrir versões PostgreSQL disponíveis na sua região
aws rds describe-db-engine-versions \
  --engine postgres \
  --region $REGION \
  --query 'DBEngineVersions[*].EngineVersion' \
  --output text

# Criar a Instância RDS PostgreSQL (compatível com Free Tier)
# Use uma das versões listadas acima no comando --engine-version
aws rds create-db-instance \
  --db-instance-identifier $DB_IDENTIFIER \
  --db-instance-class $DB_INSTANCE_CLASS \
  --engine postgres \
  --master-username $DB_MASTER_USER \
  --master-user-password $DB_PASSWORD \
  --allocated-storage 20 \
  --storage-type gp3 \
  --vpc-security-group-ids $SG_ID \
  --publicly-accessible \
  --backup-retention-period 1 \
  --no-storage-encrypted \
  --region $REGION

# Monitorar o status de criação (execute em outro terminal)
watch -n 10 "aws rds describe-db-instances --db-instance-identifier $DB_IDENTIFIER --region $REGION --query 'DBInstances[0].[DBInstanceStatus, Endpoint.Address]' --output text"
```

> **Nota:** A criação do RDS pode levar 5-10 minutos. Enquanto espera, você pode consultar o status em paralelo.
> **Free Tier:** Este comando é compatível com AWS Free Tier. Se receber erro de `backup-retention-period`, reduza para `1` ou remova a flag `--storage-encrypted`.
> **Versões:** Execute primeiro o comando `aws rds describe-db-engine-versions` para listar versões disponíveis. Se preferir uma versão específica, adicione `--engine-version VERSION` no comando create-db-instance.

3.  **Obter Endpoint e Informações de Conexão:**
```bash
# Após a instância estar "available", obtenha as informações de conexão
aws rds describe-db-instances \
  --db-instance-identifier $DB_IDENTIFIER \
  --region $REGION \
  --query 'DBInstances[0].[DBInstanceIdentifier, DBInstanceStatus, Endpoint.Address, Endpoint.Port, MasterUsername]' \
  --output table
```

*(**Conclusão:** Observe como o comando `create-db-instance` encapsula a engine (PostgreSQL), o tipo de instância (t3.micro), o armazenamento (20 GB gp3), a segurança (VPC + Security Group) e as credenciais em uma única execução.)* 

3.  **Inspeção (Se existirem DBs):** Se você já possui alguma instância RDS em sua conta, inspecione seus detalhes.
```bash
aws rds describe-db-instances \
--query 'DBInstances[0].[DBInstanceIdentifier, DBInstanceStatus, Endpoint.Address]'
```
*(**Resultado Esperado:** Visualizar o identificador, o status e o endereço de *endpoint* do DB — a informação que sua aplicação precisaria para se conectar.)*

## Seção 3: Conectando ao RDS via Terminal

1. Instalar cliente PostgreSQL (se ainda não estiver instalado):
```bash
sudo apt update
sudo apt install postgresql-client -y
```

2. Conectar ao banco usando psql (substitua com seus valores reais):
```bash
# Obter o endpoint da instância criada
ENDPOINT=$(aws rds describe-db-instances \
  --db-instance-identifier devopsdb-postgres \
  --region us-east-2 \
  --query 'DBInstances[0].Endpoint.Address' \
  --output text)

echo "Endpoint: $ENDPOINT"

# Conectar ao banco (será solicitada a senha)
psql \
    --host=$ENDPOINT \
    --port=5432 \
    --username=postgres \
    --dbname=postgres
```

3. Dentro do psql, execute comandos úteis:
```sql
-- Listar databases
\l

-- Conectar a um database específico
\c nome_database

-- Listar tabelas
\dt

-- Ver versão do PostgreSQL
SELECT version();

-- Sair do psql
\q
```

4. Ou execute queries diretamente do terminal (sem entrar no modo interativo):
```bash
psql \
    --host=$ENDPOINT \
    --port=5432 \
    --username=postgres \
    --dbname=postgres \
    --password \
    -c "SELECT version();"
```
## Seção 4: Limpeza

1. Apague a instância RDS PostgreSQL criada:
```bash
aws rds delete-db-instance \
  --db-instance-identifier devopsdb-postgres \
  --skip-final-snapshot \
  --delete-automated-backups \
  --region us-east-2
```

> **Nota:** Use `--create-db-snapshot` em vez de `--skip-final-snapshot` se desejar criar um snapshot para backup antes de deletar.

2. Aguarde a instância ser completamente removida (pode levar alguns minutos):
```bash
# Monitorar status de remoção
aws rds describe-db-instances \
  --db-instance-identifier devopsdb-postgres \
  --region us-east-2 \
  --query 'DBInstances[0].DBInstanceStatus' \
  --output text
```

3. Apague a pasta de trabalho:
```bash
cd ~
rm -rf ~/aulas_lab/aula011
```

## Dicas Importantes

1. **S3 é object storage** - ideal para arquivos estáticos e backups
2. **RDS é gerenciado** - AWS cuida de patches, backups e alta disponibilidade
3. **Nomes de bucket são globais** - devem ser únicos em toda a AWS
4. **Use VPC e security groups** - sempre configure segurança adequada
5. **Monitore custos** - RDS e S3 geram custos, sempre limpe recursos de teste
