## TF - Tarefa Final - Aula 11

### Disciplina: Implementação de servidor e nuvem (cloud)
### Aula: 11 - Armazenamento e Banco de Dados na AWS
### Objetivo do TF: Avaliar a compreensão dos tipos de armazenamento da AWS (S3, EBS, EFS) e as vantagens e desvantagens de usar serviços gerenciados (RDS) versus autogerenciados (DB em EC2).

### Orientações:

1.  Responda às questões teóricas com base no conteúdo da aula.
2.  A Tarefa Prática exige a simulação de comandos e a descrição de um cenário.
3.  As respostas devem ser objetivas e focadas na terminologia e casos de uso da AWS.

## Questão 1: Armazenamento de Objetos (S3) (Teórica)
O Amazon S3 (Simple Storage Service) é um serviço fundamental na AWS.

a) Qual é o principal caso de uso para o S3 em um contexto de aplicação Web e DevOps? (Ex: Hospedagem de sistema operacional, Logins, etc.). 
b) O S3 é um serviço global ou regional? Qual característica do S3 (durabilidade ou disponibilidade) é expressa pela taxa "Onze Noves" (99.999999999%)?

## Questão 2: Armazenamento de Blocos vs. Arquivos (EBS/EFS) (Teórica)
Existem diferentes tipos de armazenamento para diferentes necessidades de I/O (Input/Output).

a) Qual é a diferença fundamental entre o **Amazon EBS (Elastic Block Store)** e o **Amazon EFS (Elastic File System)** em termos de conexão e uso por instâncias EC2?
b) No contexto de um servidor de aplicação rodando em EC2, qual desses dois serviços (EBS ou EFS) é o mais adequado para armazenar o **Sistema Operacional** e o **executável da aplicação**?

## Questão 3: Banco de Dados Gerenciado (RDS) (Teórica)
O RDS (Relational Database Service) é o serviço de banco de dados relacional gerenciado pela AWS.

a) Cite **duas** responsabilidades de gerenciamento de banco de dados que a AWS assume ao usar o RDS (liberando o DevOps dessa tarefa).
b) Qual é a principal desvantagem ou limitação de usar o RDS em comparação com a instalação e gerenciamento de um banco de dados (ex: MySQL) diretamente em uma instância EC2?

## Questão 4: Alta Disponibilidade no RDS (Teórica)
Para garantir a Alta Disponibilidade no RDS, a AWS oferece a opção **Multi-AZ (Multi-Availability Zone)**.

a) Descreva o que acontece quando você habilita o Multi-AZ para um banco de dados RDS (onde os dados são replicados).
b) Qual a diferença entre um *Standby* no Multi-AZ e uma *Read Replica* no RDS em termos de uso e failover?

## Questão 6: Evidências Práticas de Configuração e Criação de Banco de Dados RDS

Para esta questão, você deve fornecer evidências práticas de criação e gerenciamento de um banco de dados RDS com uma tabela de alunos. Use a AWS real ou LocalStack para executar os comandos e capture prints das saídas.

### Parte 1: Evidências de Configuração

Forneça prints (screenshots) das seguintes configurações no seu ambiente WSL/Linux:

1. **Configuração de Credenciais AWS:** Print do comando `aws configure list` mostrando as credenciais configuradas (oculte chaves sensíveis).
2. **Teste de Conectividade com RDS:** Print do comando testando a conectividade com o serviço RDS na AWS (ex: `aws rds describe-db-instances`).
3. **Instalação do Cliente MySQL:** Print do comando `mysql --version` mostrando a versão instalada (ou `psql --version` se usar PostgreSQL).
4. **Variável de Ambiente do Endpoint RDS:** Print do comando que exibe a variável de ambiente configurada com o endpoint do RDS.

### Parte 2: Exercício de Criação de RDS com Tabela de Alunos

Execute os seguintes passos na AWS real ou LocalStack e forneça prints das evidências:

1. **Criar uma Instância RDS MySQL (ou PostgreSQL):** Use o comando AWS CLI para criar uma instância RDS chamada `rds-tf011-<seu-ra>` (ex: `rds-tf011-6325128`). Forneça:
   - Print do comando `create-db-instance` com os parâmetros utilizados.
   - Print do comando `describe-db-instances` mostrando a instância criada e seu status.
   - Anote o **Endpoint** da instância para uso posterior.

2. **Conectar ao Banco de Dados:** Após a instância estar disponível (status: available), conecte-se usando o cliente MySQL/PostgreSQL:
   ```
   mysql -h <endpoint-rds> -u <usuario> -p -D <nome-banco-dados>
   ```
   Forneça print do prompt de conexão bem-sucedida.

3. **Criar Tabela de Alunos:** Execute o script SQL abaixo no banco de dados:
   ```sql
   CREATE TABLE alunos (
       id INT AUTO_INCREMENT PRIMARY KEY,
       ra VARCHAR(10) UNIQUE NOT NULL,
       nome VARCHAR(100) NOT NULL,
       email VARCHAR(100) UNIQUE NOT NULL,
       data_inscricao TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
       status ENUM('ativo', 'inativo') DEFAULT 'ativo'
   );
   ```
   Forneça print do comando de criação da tabela e da confirmação de sucesso.

4. **Inserir Dados de Exemplo:** Insira pelo menos 3 registros de alunos:
   ```sql
   INSERT INTO alunos (ra, nome, email) VALUES
   ('6325128', 'João Silva', 'joao@email.com'),
   ('6325129', 'Maria Santos', 'maria@email.com'),
   ('6325130', 'Pedro Oliveira', 'pedro@email.com');
   ```
   Forneça print do comando de inserção.

5. **Verificar Dados:** Execute o comando `SELECT * FROM alunos;` para visualizar os dados inseridos. Forneça print do resultado.

6. **Criar um Backup (Snapshot) do RDS:** Use o comando AWS CLI:
   ```
   aws rds create-db-snapshot --db-instance-identifier rds-tf011-<seu-ra> --db-snapshot-identifier snapshot-tf011-<seu-ra>
   ```
   Forneça print do comando e do resultado de confirmação.

> **Nota:** Se usar LocalStack, adicione `--endpoint-url=http://localhost:4566` aos comandos AWS CLI. Coloque todos os prints no README da sua pasta RA, descrevendo cada evidência e os passos executados.
