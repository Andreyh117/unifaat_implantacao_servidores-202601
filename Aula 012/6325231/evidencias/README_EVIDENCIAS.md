# Evidencias solicitadas - Aula 012

As evidencias podem ser adicionadas nesta pasta como prints `.png`, `.jpg` ou capturas `.txt`. Chaves, senhas e tokens devem estar ocultos.

## Lista de evidencias

1. `evidencia-01-aws-configure-list.png` - Saida de `aws configure list` com credenciais mascaradas.
2. `evidencia-02-ecr-login-succeeded.png` - Login no ECR com mensagem `Login Succeeded`.
3. `evidencia-03-docker-build.png` - Build da imagem `web-app-v1:V1.0` concluido com sucesso.
4. `evidencia-04-ecr-create-describe.png` - Criacao ou descricao do repositorio ECR.
5. `evidencia-05-docker-tag.png` - Comando `docker tag` aplicado.
6. `evidencia-06-docker-images-repo-uri.png` - Saida de `docker images | grep $REPO_URI`.
7. `evidencia-07-docker-push.png` - Push para o ECR mostrando envio das layers.
8. `evidencia-08-ecr-describe-images.png` - Verificacao remota da tag enviada.
9. `evidencia-09-kubectl-deployments.png` - Bonus EKS: deployments no namespace `app-frontend`.
10. `evidencia-10-kubectl-pods.png` - Bonus EKS: pods no namespace `app-frontend`.
11. `evidencia-11-kubectl-service.png` - Bonus EKS: service LoadBalancer.
12. `evidencia-12-curl-endpoint.png` - Bonus EKS: teste HTTP no endpoint.

## Observacao

O bonus EKS e opcional e pode gerar custo. Caso nao seja executado, basta registrar no README principal que a etapa nao foi realizada.
